# TubeBoard Asset Production System

## Source Formats

Preferred editable sources:
- Figma or Sketch for layered marketing layouts.
- SVG for icon and logo masters.
- PNG exports for distribution.
- Markdown for briefs, copy, and production specs.

Do not treat generated raster images as final source. Use them as concept references unless a designer has cleaned and layered them.

## App Icon Layers

1. `base-depth`: solid Board Black with subtle radial depth.
2. `outer-glass-shell`: internal rounded glass panel, cool edge highlight.
3. `board-slab`: raised signal black panel.
4. `board-rows`: amber/yellow departure rows with strong silhouette.
5. `train-light-dots`: tiny LED nodes, aligned to board rhythm.
6. `line-accent`: thin functional line-colour traces.
7. `highlight-shadow`: specular edge and amber bloom.
8. `mask-preview`: Apple radius preview only; source remains square.
9. `export-bg`: no-alpha flattened RGB background for App Store PNG.

## Naming

Use:

`tubeboard-[surface]-[concept-or-state]-[size]-v[version].[ext]`

Examples:
- `tubeboard-icon-liquid-glass-1024-v2.png`
- `tubeboard-appstore-live-times-1290x2796-v1.png`
- `tubeboard-social-countdown-1080x1080-v1.png`
- `tubeboard-website-hero-2400x1400-v2.png`

## Export Sizes

App icon:
- 1024, 512, 192, 180, 167, 152, 120, 87, 60, 58, 40, 32.

App Store:
- iPhone 6.7: 1290x2796.
- Optional iPad: confirm if shipping universal app.

Website:
- Hero: 2400x1400.
- Open Graph: 1200x630.
- Favicon: 32/192/512.

Social:
- Square: 1080x1080.
- Story/Reels/TikTok: 1080x1920.
- Landscape: 1600x900.
- X/LinkedIn: 1200x675.

## Versioning

- Keep current repository assets as v1/starter.
- Use v2 for Liquid Glass direction.
- Never overwrite a source file unless the old version is archived or versioned.
- Keep prompt/source notes beside generated references.

## QA Checklist

- Icon readable at 32px and 40px.
- App Store icon exported with no alpha.
- No official TfL roundel or official-looking lockup.
- Screenshots use implemented app states only.
- Copy does not imply official endorsement.
- All text passes contrast requirements.
- Marketing overlays readable at thumbnail size.
- Assets share colour, glass, and motif rules.
- File names include size and version where practical.
- Export folders contain only useful files, not build artifacts.
