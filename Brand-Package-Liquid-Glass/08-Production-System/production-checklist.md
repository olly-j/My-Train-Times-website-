# TubeBoard Production Checklist

## Before Final Asset Export

- Confirm final app name and subtitle.
- Confirm implemented launch features.
- Confirm TfL Open Data attribution wording.
- Confirm privacy policy and support URLs.
- Approve icon v2 direction.
- Capture real app screenshots from current build.
- Decide whether roadmap iOS ecosystem features appear only in docs or are launch-ready.

## Icon

- Export 1024 PNG, RGB, no alpha.
- Export full iOS size set.
- Preview at 1024, 180, 120, 60, 40, 32.
- Test on light wallpaper, dark wallpaper, and App Store search.
- Verify no roundel-like composition.

## Website

- Replace hero/OG rasters with v2 visual system.
- Update final App Store link.
- Add privacy/support links.
- Check responsive layouts at 390, 768, 1440 widths.
- Check contrast and keyboard focus.
- Respect reduced motion.

## App Store

- Use real screenshots only.
- Keep headlines readable in thumbnail.
- Include independent/data-source trust screenshot if possible.
- Keep roadmap features out of final screenshots unless implemented.
- Prepare review notes.

## Social/Press

- Export square/story/landscape launch graphics.
- Prepare press screenshots and icon ZIP.
- Add founder/developer note.
- Confirm reviewer access path.

## Packaging

- Keep source and exports separated.
- Include README or index of deliverables.
- Remove temporary files.
- Zip only useful docs/assets.
- Run `git status` and review all changes before handoff.
